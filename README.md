# QR Учёт товаров (Django)

Система учёта товаров с авто-генерацией ID и QR-кода, ролями
«Владелец / Кассир», учётом остатков и историей продаж.

## Что внутри

- **Регистрация**: любой человек может зарегистрироваться сам — указывает
  никнейм (логин), email (только @gmail.com), пароль и подтверждение
  пароля. Этот аккаунт становится **владельцем** — все товары, кассиры
  и продажи привязаны к нему.
- **Кассиры**: владелец может создавать аккаунты кассиров (логин/пароль).
  Кассир видит те же товары, что и владелец, и может оформлять продажи,
  но не может добавлять/удалять товары и не видит раздел «Кассиры».
- **Товар**: название, цена (необязательно), количество на складе,
  авто-ID (например `A83K29`), QR-код (PNG).
- QR хранит данные в формате `ID|Название|Цена`.
- **Продажа**: кнопка «Продать» — указываете количество, остаток
  уменьшается, запись попадает в историю продаж (раздел «Продажи»).
- **Пополнение склада**: владелец может добавить товар на склад («+ Пополнить»).
- Список товаров с превью QR, кнопками «Скачать» и «Печать».
- Хранение: SQLite (`db.sqlite3`), QR-картинки — в папке `media/qr_codes/`.

## Установка и запуск

1. Создайте виртуальное окружение и установите зависимости:

```bash
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

2. Примените миграции (создаст файл `db.sqlite3`):

```bash
python manage.py migrate
```

> Если вы уже запускали более раннюю версию проекта и у вас уже есть
> `db.sqlite3` со старой структурой таблиц — удалите файл `db.sqlite3`
> (и при желании содержимое `media/qr_codes/`) перед `migrate`, так как
> структура таблиц изменилась (добавились роли, владелец, остатки и продажи).

3. (Опционально) Создайте суперпользователя для доступа в админку Django:

```bash
python manage.py createsuperuser
```

4. Запустите сервер разработки:

```bash
python manage.py runserver
```

5. Откройте в браузере: http://127.0.0.1:8000/
   Вас перекинет на страницу входа. Нажмите «Зарегистрироваться», введите
   никнейм, email (@gmail.com) и пароль — это и будет ваш аккаунт владельца.

## Как это работает

### Владелец
1. Регистрируется (никнейм + email @gmail.com + пароль).
2. На странице «Товары» нажимает «+ Добавить товар»: вводит название,
   цену и начальное количество. ID и QR создаются автоматически.
3. На странице «Кассиры» может создать одного или нескольких кассиров
   (логин + пароль) — они будут видеть те же товары и продажи.
4. Может пополнять склад («➕ Пополнить») и удалять товары.

### Продажа
1. На карточке товара нажимается «🛒 Продать».
2. Указывается количество — система проверяет, что на складе достаточно.
3. После подтверждения: остаток товара уменьшается, в «Продажи»
   добавляется запись (товар, количество, цена, сумма, кто продал, когда).

### Кассир
- Входит под логином/паролем, которые создал владелец.
- Видит товары и остатки, может оформлять продажи и смотреть историю продаж.
- Не видит кнопки «Добавить товар», «Пополнить», «Удалить» и разделы «Кассиры» / «Настройки».

### Настройки → Опасная зона (только для владельца)
- **Очистить мои данные** — удаляет все ваши товары (с QR-кодами) и всю
  историю продаж. Аккаунт и кассиры остаются.
- **Удалить мой аккаунт** — после ввода пароля удаляет ваш аккаунт целиком,
  все товары, продажи и аккаунты ваших кассиров. Действие необратимо.

Оба действия касаются только данных текущего владельца — на других
пользователей не влияют.

### Удаление любых аккаунтов через суперпользователя
Django-админка (`/admin/`) в этом проекте отключена — обычные
пользователи о ней даже не узнают. Управление через суперпользователя
происходит прямо на сайте: войдите под аккаунтом, у которого
`is_superuser=True` (создаётся командой `createsuperuser`), и в верхнем
меню появится кнопка «Сброс системы» (см. ниже). При удалении любого
аккаунта владельца (через свой аккаунт → «Удалить аккаунт» или через
«Сброс системы») автоматически удаляются и все его кассиры — товары и
продажи удаляются вместе с записью благодаря связям в базе.

### Кнопка «Сброс системы» (только для суперпользователя)
Если вы вошли на сайт под суперпользователем, в верхнем меню появится
красная ссылка **«Сброс системы»**. После ввода пароля она удаляет:
- все аккаунты владельцев и кассиров (кроме вашего суперпользователя),
- все товары и QR-коды,
- всю историю продаж.

После этого система выглядит как только что установленная — можно
заново регистрировать владельцев. Сам файл `db.sqlite3` и его структура
не трогаются, удаляются только данные (это программный аналог
`python manage.py flush`, но с сохранением вашего суперпользователя).

## Возможные доработки

- Сканер QR через камеру (например, библиотека `html5-qrcode`) для
  быстрой продажи по сканированию.
- Категории товаров, фото товара.
- Массовая печать QR-кодов на листе A4 (PDF).
- Поиск/фильтрация товаров по названию или ID.
- Отчёты по продажам за период, по кассирам.

Если что-то из этого нужно — просто скажите, добавим.

## Настройка Email (подтверждение при регистрации)

1. Зайди в Google аккаунт → Безопасность → Двухфакторная аутентификация (включи)
2. Там же → "Пароли приложений" → создай новый (тип: Почта) → скопируй 16 символов
3. Открой `qr_inventory/settings.py` и замени:

```python
EMAIL_HOST_USER = 'your@gmail.com'       # ← твой Gmail
EMAIL_HOST_PASSWORD = 'your-app-password' # ← 16-значный пароль приложения
DEFAULT_FROM_EMAIL = EMAIL_HOST_USER
```

После этого при регистрации будет отправляться 6-значный код на указанный email.
Код действует 10 минут.
